Project Overview

This is a conceptual digital wallet app created to explore how UX design can reduce confusion and improve trust during everyday financial transactions. The goal was to demonstrate research-driven UX thinking rather than build a live product.

Why This Problem?

Many users face anxiety due to unclear transaction status, failed payments, and cluttered wallet interfaces. This project focuses on designing a calm, simple, and reliable wallet experience.

Project Goals

  • Simplify the payment experience
  • Reduce steps in transaction flows
  • Improve transaction status clarity
  • Build user confidence and trust

User Research

Since the product is not live, qualitative research was conducted to understand real user behavior.

Research Methods
  • 1-on-1 user interviews (12 users)
  • Online survey (60+ responses)
  • Competitive analysis of existing wallet apps
Target Users
  • Age group: 20â€“45
  • Office employees, students, freelancers
  • Regular wallet app users

Key Research Insights

  • Users check balance before making payments
  • Transaction status clarity is more important than visuals
  • Too many options cause hesitation
  • Processing states create anxiety

User Pain Points

Pain Point Impact
Overloaded home screen Confusion and slower decisions
Unclear transaction status Anxiety and lack of trust
Long payment flow User drop-offs
Complex terminology Low understanding
Hidden transaction history Poor transparency

Problem Statement

Users need a simple and transparent wallet experience that allows them to complete payments confidently without confusion or fear of failure.

Design Solution

The solution focused on removing friction instead of adding features, with emphasis on clarity, minimal steps, and immediate feedback.

Design Principles
  • Clarity over complexity
  • Minimal steps for key actions
  • Immediate and clear feedback
  • Human-friendly language

Outcome

Although conceptual, this project demonstrates strong UX fundamentals and shows how thoughtful design can improve trust in fintech products.

Key Learnings

  • Simplicity builds trust
  • Clarity is critical in financial UX
  • Small UX decisions influence user confidence
